# 更新日志 (Changelog)

本项目的所有变更都会记录在这里。

## [2.1.0] - 2026-04-20

这个版本大幅增强了输出的可控性与工程闭环能力，正式引入自动化指令系统 (PulseCommand)，并针对大模型对话场景优化了上下文结构。

### 主要更新
- **自动化指令系统**：新增“自动化指令”设置项，引导 AI 输出规范化代码修改指令，用户可直接在新增的指令控制台中运行指令以应用改动，打通了从“建议”到“落地”的自动化流程。
- **输出控制增强**：
  - **结构大纲模式**：新增“仅输出结构大纲”设置，开启后将忽略具体文件内容。
  - **行号输出支持**：新增“显示行号”设置项，生成的内容将包含原始代码行号，方便 AI 进行精准的代码定位和修改。
- **配置交互升级**：设置弹窗底层组件升级，支持动态宽高调整，并增强了表单控件库（如新增 Select 控件）。

### 修复与优化
- **解析引擎优化**：优化了后端 Rust 依赖解析算法及项目路径解析逻辑，提升了在复杂工程下的鲁棒性。
- **API 服务稳定性**：修复了 API 服务启动时可能出现的端口占用 (os error 10048) 错误；完善了 Swagger UI 的 Schema 示例。
- **逻辑缺陷修复**：解决了提示词（Prompt）修改后未能及时触发自动生成的同步问题。
- **体验微调**：固定了依赖树视图的顶部状态栏高度，保证了在长列表滚动时的交互一致性。

---

## [2.0.1] - 2026-03-28

新增 API 服务与自动更新功能两个大功能。优化了上下文结构与界面交互体验。

### 主要更新
- **新增 API 服务**：基于本地 Hono 服务层全新打造了外部 API 模块，提供标准 RESTful 接口，方便 CLI 或第三方应用直接接入调用解析引擎。
- **集成 OpenAPI 文档**：伴随着 API 服务上线，我们还集成了 OpenAPI 规范和 Swagger 界面，方便在浏览器里直接看参数说明和测试调用。
- **自动更新功能**：原生集成 Tauri 自动更新（Updater）流程，补充自动化发布管线以及本地更新模拟验证系统。
- **设置页改版**：弹窗式设置升级为侧边栏导航形式，重新梳理设置项顺序，布局更加清晰并支持更好的控件管理。

### 修复与优化
- **精简上下文输出**：将分开的“主文件提示”和“文件依赖关系”合并，提升注意力，更省 Token。
- **文件路径展示**：新增路径简化显示的功能。
- **交互微调**：微调点击目录树后右侧代码框定位滚动的体验。

---

## [1.3.0] - 2026-03-25

这个版本进行了一次大规模的架构重构，将原本臃肿的主入口进行了逻辑拆分，并完善了自动化发布流程，显著提升了项目的稳定性和可维护性。

### 主要更新
- **架构重构**：将 `App.vue` 中的解析逻辑、拖拽处理、设置页逻辑独立拆出；并将组件库按 `layout`、`file-tree` 等职责进行分组归类。
- **自动化流水线**：集成 GitHub Actions 支持多平台（macOS/Linux）自动化打包发布。
- **重点文件提示**：新增重点文件提示开关，方便用户聚焦核心逻辑。
- **布局交互优化**：依赖文件树现在支持拖动调节宽度。

### 修复与优化
- **交互体验**：修复了点击编辑时文本框滚动位置可能跳变的问题。
- **构建修复**：修正了 `tauri.conf.json` 中的部分配置项，确保跨平台打包顺利进行。
- **性能微调**：优化了文件树节点的渲染逻辑，支持悬停显示完整路径。

---

## [1.2.0] - 2026-03-23

这次更新主要是升级了依赖分析算法。我们引入了 AST 解析机制，相比之前的正则方案，现在识别各种导入语法（import/require）更准了，兼容性也更好。

### 主要更新
- **依赖分析算法升级**：集成了 AST 解析支持，替换了部分太稳的正则提取逻辑，识别更精准。
- **支持别名导入**：现在能识别并解析带别名（Path Alias）的导入路径了。
- **层级逻辑优化**：手动上传整个目录时，内部文件层级统一视为 0，方便统一处理。
- **体验改进**：项目树中的节点在鼠标悬停时会显示完整名称。

### 修复与优化
- **修复 Bug**：解决了 TS 中部分不带后缀的导入语句无法识别的问题。
- **多语言优化**：修复了部分非 TS/JS 语言在解析依赖时可能出现的异常。
- **代码清理**：移除了前端一些没用的缓存清理逻辑。

---

## [1.1.0] - 2026-03-22

这个版本重点优化了后端的依赖解析能力，增加了缓存机制，解析速度比之前快了不少。

### 主要更新
- **增加缓存机制**：引入了简单的后端文件缓存，重复解析同一个项目时速度飞快。
- **支持随时中断**：解析大项目如果等不及了，现在可以手动点中断，不会死在那儿。
- **Vue 自动引入支持**：优化了匹配逻辑，现在能识别 Vue 项目里那些自动引入的组件依赖了。

### 修复与优化
- **后端重构**：把臃肿的 `analyzer.rs` 拆成了多个小模块（`deps.rs`, `resolve.rs` 等），代码更清爽了。
- **设置页优化**：面板现在能记住你上次折叠或展开的状态了。
- **交互细节**：增加了对非文本文件的过滤警告（拖拽时会有红框提示）；给数字输入框加了步进控制。
- **修复 Bug**：解决了“压缩触发层级”设为 0 时不生效的问题。
- **发布自动化**：写了个 `release.js` 脚本，以后发版本和交叉编译更省事了。

---

## [1.0.0] - 2026-03-22

这是 CodePulse 的第一个正式版本。主要实现了代码上下文的提取和精简，方便大家把代码喂给 AI。

### 主要功能
- **多语言依赖解析**：支持 TS/JS, Rust, Python 等 10 多种语言的递归依赖解析。
- **上下文压缩**：支持只保留类和函数的定义，把具体的实现删掉，帮你在对话时省点 Token。
- **结构化输出**：自动生成项目文件树，并把代码格式化成 AI 比较好懂的结构。
- **新的设置页**：支持自己配置过滤规则、根目录和文件类型。

### 优化与修复
- **性能改进**：用了异步任务处理，解析复杂依赖时界面不再卡死。
- **界面微调**：调整了配色和滚动条，看着更舒服点。
- **操作改进**：支持直接把文件拖进来解析。
